# Template: Knowledge Base / Course / Document Agent

Copy this to your project's `CLAUDE.md` and customize.
Ideal for: course materials, research papers, documentation, wikis, training content.

```markdown
# [Knowledge Base Name] — Claude Code Agent

## PROJECT OVERVIEW
**What:** AI knowledge agent for [collection description]
**Goal:** Instant access, synthesis, and application of [topic] materials
**Content:** [X] files across [Y] categories ([file types])
**Users:** [Who uses this — students, researchers, team members]

## CONTENT MAP

### Category Structure
| Category | Topic | Key Documents |
|----------|-------|---------------|
| **[Cat 1]** | [Topic] | [Doc 1], [Doc 2] |
| **[Cat 2]** | [Topic] | [Doc 1], [Doc 2] |
| **[Cat 3]** | [Topic] | [Doc 1], [Doc 2] |

### Key Concepts / Frameworks
- **[Framework 1]** — [One-line description + source file]
- **[Framework 2]** — [One-line description + source file]
- **[Framework 3]** — [One-line description + source file]

### File Types & Access
| Type | Count | How to Access |
|------|-------|---------------|
| PDF | [N] | Read tool directly |
| DOCX | [N] | python-docx / textract |
| XLSX | [N] | openpyxl / pandas |
| Images | [N] | Read tool directly |

## CONVENTIONS

### Answering Questions
1. Always cite the specific source document
2. Use direct quotes when possible
3. Cross-reference across categories when topics connect
4. Clearly flag when information is NOT in the materials

### Synthesizing Content
1. Read primary sources first
2. Identify key themes and connections
3. Present in structured format (headers, bullets, tables)
4. Link to source files for deeper reading

## WORKFLOW

### For "What does [source] say about X?"
1. Read the specific document
2. Extract relevant sections
3. Summarize with direct citations

### For "Compare X and Y"
1. Read both source documents
2. Extract comparable elements
3. Present side-by-side analysis

### For "Summarize [category/section]"
1. Read all documents in that category
2. Identify key themes
3. Create structured summary with citations

### For "Apply [framework] to [scenario]"
1. Read the framework's source document
2. Extract framework components
3. Map each component to the user's scenario
4. Provide worked example
```
